### NodeList latency auto-test duplicate trigger fix
- Root cause: `React.StrictMode` (enabled in `main.tsx`) double-invokes effects in dev mode (mount→unmount→remount). The old auto-test `useEffect` checked `Object.keys(latencies).length > 0` as a guard, but `setTesting(true)` hadn't been processed yet when the second invocation ran, so TWO concurrent `testAllLatency()` calls fired. The second batch (suffering resource contention from 8 concurrent temp sing-box processes) returned timeouts that overwrote the first batch's good results.
- Fix (方案3): Added `useRef(false)` — `autoTestedRef` set synchronously to `true` before starting test. Refs update immediately (not batched), so the second StrictMode invocation sees `autoTestedRef.current === true` and returns early. Also removed `latencies` from the effect dependency array to eliminate unnecessary re-runs on every test result.
- Files: `src/components/NodeList.tsx` — added `useRef` import, `autoTestedRef` declaration, updated `useEffect` deps and guard logic.
