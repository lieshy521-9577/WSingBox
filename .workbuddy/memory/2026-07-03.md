### TUN mode UAC optimization (Option A + Option B combined)

**Problem**: TUN mode connection triggered up to 3 UAC prompts (elevated start, elevated stop, re-elevated start for bootstrap retry).

**Solution тАФ Option A (Merge elevated ops)**:
- `launch_singbox_with_config(tun=true)` now includes `Stop-Process -Name 'sing-box' -Force -ErrorAction SilentlyContinue` at the start of the elevated PowerShell script тАФ kills any existing sing-box + starts new one in a single `Start-Process -Verb RunAs` call (1 UAC).
- Bootstrap retry paths in `start_singbox` no longer call `stop_singbox_process()` separately; instead just call `launch_singbox_with_config()` which handles kill+start in one elevated command. Replaced `core_process::stop_singbox_process()` with `core_process::clear_core_state()` + `core_process::clear_core_pid()` + direct launch.
- `stop_singbox_process()` now checks `is_elevated()` тАФ if already admin, skips the elevated kill fallback entirely (direct kill works for admin processes).

**Solution тАФ Option B (Self-elevation)**:
- Added `is_elevated()` in `core_process.rs` тАФ checks Windows Administrators group SID via PowerShell.
- Added Tauri commands: `is_elevated`, `request_elevation`, `check_elevation_intent`.
- `request_elevation`: saves elevation intent to disk, spawns elevated instance of the app via `Start-Process -Verb RunAs`, exits current instance.
- `check_elevation_intent`: reads one-shot intent flag and deletes it. Frontend calls on startup, auto-connects if flag is true.
- `launch_singbox_with_config(tun=true, is_elevated=true)` spawns sing-box directly with `Command::new` тАФ no UAC needed at all.
- Frontend: added `isElevated` and `requestElevation` to `useSingbox.ts` hook. Sidebar shows elevation status banner:
  - Not elevated: amber "Restart as admin for 0 UAC prompts" button
  - Elevated: emerald "Running as admin тАФ no UAC needed" badge
- Elevation intent auto-connect: after admin restart, `useEffect` checks `check_elevation_intent` and calls `startProxy()` if true.

**Result**: 
- Best case (app running as admin): 0 UAC prompts for all operations
- Normal case (not admin, TUN mode, clean start): 1 UAC prompt (combined kill+start)
- Worst case (not admin, TUN + bootstrap retry): 1 UAC prompt (combined kill+restart)
- Old worst case was 3 UAC prompts
