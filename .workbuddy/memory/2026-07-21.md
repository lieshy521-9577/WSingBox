**Bug found**: `release.ps1` failed at SHA-256 step. Root cause:
- User ran `release.ps1 -Version v1.4.0` (with `v`)
- Script used `$Version` directly in NSIS path → looked for `SingBox Client_v1.4.0_x64-setup.exe`
- But Tauri NSIS output uses RAW version from tauri.conf.json → `SingBox Client_1.4.0_x64-setup.exe` (no `v`)
- Portable zip was built twice (once `1.4.0`, once `v1.4.0` from failed run)

**Fix** (commit `efaf471`):
- `release.ps1`: normalize `$ver = $Version -replace '^v', ''`, use for filenames, keep `v` only for git tag/release title
- `package-portable.ps1`: same normalization for standalone calls
